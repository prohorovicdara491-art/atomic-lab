## 4. Ответы на контрольные вопросы

**1. Как git add -p помогает при отладке через git bisect?**  
`git add -p` позволяет разбить изменения на атомарные коммиты. При `git bisect` это помогает точно определить коммит, вызвавший ошибку.

**2. Почему requirements.lock.txt критичен для командной работы?**  
Фиксирует точные версии **всех** зависимостей, обеспечивая одинаковое окружение у всех разработчиков и на серверах.

**3. Преимущество Makefile перед текстовой инструкцией?**  
Makefile — исполняемая документация. Автоматизирует команды, исключает ошибки, унифицирует процесс.

**4. Тесты как живая документация и фиксация seed?**  
Тесты Given-When-Then показывают использование кода. `random.seed(42)` делает тесты детерминированными.

**5. Что будет при make install на чистой машине?**  
Создаст новое виртуальное окружение и установит зависимости из lock-файла. Приложение будет работать идентично.
