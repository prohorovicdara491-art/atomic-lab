# Анализ рисков подхода с нарушением атомарности

## 1. Сложность отката (revert)
Когда в одном коммите содержатся разные типы изменений (фичи, фиксы, рефакторинг), невозможно откатить только одну часть изменений. Например, если новая функция вызывает баг, мы не можем откатить только её, не потеряв важные исправления.

## 2. Проблемы при код-ревью
Смешанный коммит сложно ревьюить. Рецензент должен одновременно анализировать:
- Изменения бизнес-логики
- Исправления ошибок
- Рефакторинг кода

Это увеличивает когнитивную нагрузку и повышает риск пропустить ошибку.

## 3. Риск «снежного кома» зависимостей
requirements.txt без указания версий может привести к:
- Непредсказуемым обновлениям пакетов
- Различиям в окружениях у разных разработчиков
- Неожиданной поломке кода после переустановки зависимостей
